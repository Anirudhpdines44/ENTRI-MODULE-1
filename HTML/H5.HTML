Introduction

HTML5 is a markup language used for structuring and presenting content on the World Wide Web. It is the fifth and current major version of the HTML standard. It aims to improve the language with support for the latest multimedia, while keeping it easily readable by humans and consistently understood by computers and devices.

HTML5 is intended to subsume not only HTML 4, but also XHTML 1, and DOM Level 2 HTML. HTML5 includes detailed processing models to encourage more interoperable implementations; it extends, improves, and rationalizes the markup available for documents, and introduces markup and application programming interfaces (APIs) for complex web applications.

New Features

HTML5 introduced many new features designed to make web development easier and more powerful. Key additions include:

  • New semantic elements .
  • Native multimedia support tags.
  • Graphics with and SVG.
  • Enhanced form controls.
  • New JavaScript APIs for geolocation, local storage, drag and drop, and more.

Semantic Elements

The HTML5 Badge from W3C

One of the most significant changes in HTML5 is the introduction of **semantic elements**. These elements provide more meaning about the content they contain, which helps both search engines and assistive technologies better understand the structure of a webpage. Instead of relying solely on generic `

` elements, developers can now use tags that clearly define the purpose of a section of content.

Examples include:

  • <article>: Self-contained content, like a blog post or news story.
  • <section>: A thematic grouping of content, typically with a heading.
  • <nav>: Navigation links.
  • <header>: Introductory content for a section or the whole document.
  • <footer>: Concluding content, often copyright information or contact details.

Multimedia Support

New APIs

HTML5 also introduced a range of new JavaScript APIs that enable more dynamic and interactive web applications. Some notable APIs include:

  • **Geolocation API**: Allows web applications to access the user's location.
  • **Web Storage API**: Provides mechanisms for web applications to store data locally within the user's browser (localStorage and sessionStorage).
  • **Drag and Drop API**: Enables elements to be dragged and dropped within a webpage.
  • **Canvas API**: Provides a means for drawing graphics and animations on a web page using JavaScript.

These APIs empowered developers to create rich, desktop-like experiences directly in the browser without relying on external plugins.

History

The development of HTML5 was started by the Web Hypertext Application Technology Working Group (WHATWG) in 2004, initially as a platform for web applications. It was later adopted by the World Wide Web Consortium (W3C) in 2007. The first public draft was released in 2008, and it achieved W3C Recommendation status on 28 October 2014. The development continues with subsequent versions such as HTML 5.1 and HTML 5.2.
